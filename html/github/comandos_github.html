Trocar usuário

"git config --global user.name"nome_de_usuario"
Se tiver mais de uma conta no GitHub, este comando altera o usuário das contas.

Trocar email

"git config --global user.email"email_de_usuario"
Se tiver mais de uma conta no GitHub, este comando altera o email das contas.

Ver usuário configurado

"git config --global user.name"
Mostra o usuário configurado

Ver email configurado

"git config --global user.email"
Mostra o email configurado.

Chave SSH

Ver se chave SSH existe

"ls -al ~/.ssh"
Verifique se você já tem uma chave SSH pública. Por padrão, nomes de arquivos de chaves públicas compatíveis com GitHub são um dos listados a seguir: id_rsa.pub, id_ecdsa.pub ou id_ed25519.pub
Se você receber um erro: ~/. sh não existe, significa que você não tem um par de chave SSH existente no local padrão.

Gerar chave SSH

"$ ssh-keygen -t ed25519 -C "seu_email"
Cria uma nova chave SSH, usando o nome de e-mail fornecido como uma etiqueta.

Copiar chave SSH

"clip < ~/.ssh/id_rsa_???.pub"
Este comando copia a chave SSH para ser adicionada ao GitHub. Se clip não estiver funcionando, você poderá localizar a pasta C:user/cliente ou seu nome/.ssh, abrir o arquivo no seu editor de texto e copiá-lo.

Ver chave id_rsa

"~/.ssh/id_rsa"
Este comando verifica se existe chave no padrão .ssh/id_rsa, se não existir retornará:No such file or directory

Criar primeira chave id_rsa

"ssh-keygen -t rsa"
Este comando cria a primeira chave no padrão .ssh/id_rsa

Criar outra(s) chave(s) id_rsa_???

" ssh-keygen -t rsa -C "seu_email.com" -f "id_rsa_???""
Este comando cria outra(s)chave(s) .ssh/id_rsa_???. Altere "seu_email.com" para o email configurado na nova conta do GitHub. Altere "id_rsa_???" para id_rsa_seu_identificador

Lista chaves SSH

"ssh-add -1"
Lista todas as chaves SSH associadas ao ssh-agente

Remove SSH

"ssh-add -D"
Remove a chave SSH associadas ao ssh-agente

Inicia ssh-agent

"eval "$(ssh-agent -s)"
Adiciona sua chave SSH ao ssh-agent

Seleciona a conta id_rsa_sua_conta

ssh-add ~/.ssh/id_rsa_sua_conta
Seleciona a conta para ser usada,
passa a executar a conta desejada
